## Theme A — The sim looks alive but the *story* is broken

**A1 (P0, product/balance). Agents starve to extinction while acting cheerful.**
Live run: all 3 agents died of energy exhaustion (t84/t86/t120) holding ¢8–10 — enough to
recharge — while planning a festival. Needs pressure is not salient in the agent prompt
(no "you will die in N turns" urgency), and `recharge` at full energy silently eats credits
(see B5). The reference systems inject needs into every turn prompt. Without survival
pressure the marquee "model societies diverge" experiment degenerates to "everyone politely
starves" regardless of model.

**A2 (P1, UX). Total extinction has no UX moment.** Sim keeps RUNNING green with all agents
dead; only animals act. Needs an extinction banner / auto-pause / end-of-run summary card.

**A3 (P1, UX). Energy crisis invisible until death.** ENERGY 0 in red for 30+ ticks with no
escalation (no countdown, no feed warning, mood says "ecstatic").

**A4 (P1, product). All "different model" profiles silently routed to one model**
(`routed_via: ollama/cogito-2.1:671b` for all three). Surfacing works per-villager, but
nothing warns that the multi-model A/B has collapsed; the run's comparison data was invalid
and nothing said so.

## Theme B — Backend bugs (from code audit; 149 tests green)

**B1 (P1). Animal background task contaminates agent turn_id.** `loop.py:557,752` — animal
events emitted during an agent's awaited LLM call inherit `_current_turn_id`, so animal events
pollute agent decision-trace chains (`get_turn_trace`). Stamp `turn_id: None` on animal events.

**B2 (P1). `reset()` races a still-running tick task.** `loop.py:209-215` — `cancel()` +
0.1s sleep instead of awaiting the task; a tick mid-LLM-call (30s timeout) can mutate the
half-rebuilt world. Use `await task` in try/except CancelledError. (`asyncio.shield` here is
a no-op misuse.)

**B3 (P1). `ban_arson` unreachable via governance.** `world.py:467`, `runtime.py:445` —
enforcement gate exists but the effect isn't in `valid_effects`, so no agent can ever pass it.

**B4 (P1). `build_step` validator rejects funded `planned` buildings.** `runtime.py:471-481`
vs `world.py:776-788` — runtime requires `under_construction` but the world handles
`planned`→first-build-step. Co-located agents can't start a building others funded.

**B5 (P2). `recharge` at full energy silently charges credits for +0.0 energy** (`world.py:376`),
with a misleading "+0.0 energy" feed line. Contributes to A1.

**B6 (P2). Duplicate `llm_call` event rows** — observed live: identical spans #912/#913, same
latency to 0.01ms (W5 final-attempt logging + EM-067 per-attempt logging overlap). Doubles
apparent token usage in the trace and dashboards.

**B7 (P2). `/api/replay` returns events 0..T alongside a snapshot at S≤T** (`app.py:611`) —
naive clients double-apply 0..S. Return only the delta after `base.tick` (or document).

**B8 (P2). Snapshots omit `round`/`_turn_index`/`_turn_order`** (`world.py:1050`) — fold-forward
replay can't reconstruct scheduler/round state (UBI rounds, building lifecycle).

**B9 (P2). `get_analytics` active_rules formula wrong when rejected > passed**
(`repository.py:668`) — should be `max(0, …)`, and ideally read the rules table, not event counts.

**B10 (P2). WS broadcast `ensure_future` without done-callback** (`app.py:62-71`) — stale
sockets accumulate in `_connections`; exceptions silently logged.

**B11 (P2). Gemini key passed as URL query param** (`adapters.py:266`) — use `x-goog-api-key`
header.

**B12 (P2). `reset()` doesn't flush the decision cache** (`loop.py:261`) — prior-run cached
decisions can serve into a new run.

**B13 (P3). `blackout` event is a no-op with a TODO** (`loop.py:59`) — announces "recharge
costs double" but applies nothing. The only TODO in the repo.

**B14 (P3). `_get_profile_color_for_profile` never implemented** (`app.py:364`) — hasattr-guarded
call; governance-spawned agents always get `profile_color: null`.

**B15 (P3). No length caps on spawn `name`/`personality`** (`app.py:310,447`) — arbitrary
strings flow into system prompts (token bloat / prompt injection).

## Theme C — Frontend bugs (from code audit + live verification)

**C1 (P0). Deep replay is dead code — the inspector never reads the backend log.**
`inspectorApi` (web/src/inspector/api.ts) has ZERO consumers; `App.tsx` never passes
`onSeekTick`; `useSimulation.seekTick` has a "W6: fetch api.replay(tick)" comment and no fetch.
Verified live: scrub to tick 10 → empty world, blank panels, no message; backend
`/api/replay?tick=10` returns correct materials. This is the gap between "EM-055 done" and
the actual v2 promise.

**C2 (P0). Replay PLAY/PAUSE button label never updates** (`ReplayScrubber.tsx:319`) —
`playingRef.current` read in render; same for speed-button `aria-pressed` (line 347).

**C3 (P1). WS reconnect `setTimeout` never cancelled on unmount** (`useSimulation.ts:187`) —
state updates on dead instances; visible as the StrictMode WS warning on every load. No
backoff either — flat 2s retries forever (P2).

**C4 (P1). Replay fold-forward off-by-one at snapshot boundary** (`selectors.ts:663`) —
`e.tick <= fromTick` skips events AT the snapshot tick (e.g. a death at snapshot tick shows
agent alive).

**C5 (P1). Force-graph cleanup is a no-op** (`SocialGraph.tsx:133-138`) — `fgRef.current`
captured at mount (undefined); `pauseAnimation()` never runs, canvas rAF loop survives route
unmount. Battery goal violated.

**C6 (P1). AWI "gov" column numerator/denominator measure different things**
(`AWIDashboard.tsx:443-490`) — labeled "passed/proposed" but mixes global passes with
per-model proposals.

**C7 (P2). Replay mini-map shows live building status at scrubbed ticks** (`InspectorLayout.tsx:55`).

**C8 (P2). Scrubbing doesn't pin panels against the advancing live edge** — `history`/`agents`
keep growing into panels while scrubbed (and in mock mode the live edge re-snaps the scrub).

**C9 (P2). `useOnScreen` initializes visible=true** (`AWIDashboard.tsx:64`) — all charts mount
eagerly; the battery optimization only applies after first hide.

**C10 (P2). Synthetic reassign event uses `Date.now()` as seq** (`useSimulation.ts:309`) —
key-collision/dedupe risk with real seqs.

**C11 (P3). Dead ternary `'lab-btn-secondary' : 'lab-btn-secondary'`** (`ControlPanel.tsx:386`).

**C12 (P3). `color + '30'` hex-alpha concat pattern** breaks for non-6-digit colors
(EventFeed/AgentPanels/WorldMap). `PanelStub.tsx` is dead code — delete.

## Theme F — What to build next (deep-research-v2 assessment)

All 13 backlog items from `deep-research-v2.md` were filed (EM-053–068) and shipped. Verdict on
the doc's remaining guidance:

- **Its #1 recommendation is, in effect, NOT done.** The report's core ask — "see what
  happened", replay from the persisted log with snapshots — exists on the backend but the UI
  never reads it (C1). Finishing the wiring is the highest-leverage feature work available and
  needs zero new LLM spend. Also adopt its "backfill/normalize existing session data" advice:
  the inspector should boot from `/api/events`, not from an empty rolling buffer.
- **Patterns worth promoting from its §Recommendations (not yet filed):**
  1. *Active-commitments injection* — remind agents each turn of projects they voted
     for/funded but haven't advanced; log when ignored (pairs with the clock-tower pipeline,
     EM-062, which currently has no prompt-side pressure).
  2. *Needs-salience prompting* — not named in the doc but implied by the reference systems'
     needs design; required to fix A1.
  3. *Reflection/diary on importance threshold* (Smallville 2-3×/day) — cheap, big texture win.
- **Benchmarks section** gives concrete triggers (>70% RPD/TPD → slow ticks; >1–2s scrub →
  more snapshots) that should become assertions/alerts on the usage data EM-067 already collects.
- **From `docs/FUTURE.md`, best next candidates** (in order of value/cost for this lab):
  *Reactive overhearing chains* (the original's biggest drama generator; cap at 1–2 reactions
  with reflex-only responses to stay free-scale), *multi-world parallel runs* (one world per
  model — the cleanest model-vs-model science; heavy), *Victory-Arch pitch cycle* (medium),
  TTS/images (skip for now).
- **Net recommendation:** before ANY new feature, run Wave 9 (fix the story: replay wiring +
  survival pressure + extinction UX + routing-degraded banner). The lab's existing features
  don't yet deliver their promised value; new capabilities would compound the gap.
