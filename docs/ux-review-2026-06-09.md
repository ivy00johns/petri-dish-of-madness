# UX/UI Review — PetriDishOfMadness
**Date:** 2026-06-09
**Pages reviewed:** `/` (3D village + 2D map toggle), `/inspector` (replay, decision trace, governance, social graph, AWI dashboard, animal chaos feed)
**Viewports tested:** 1280x800, 768x1024, 375x812
**Context:** reviewed against a LIVE FreeLLMAPI run (run 1, ticks ~73–121). During the review all three agents died of energy exhaustion (Ada t84, Bram t86, Cleo t120) — which exercised the death UX end-to-end and surfaced a major game-balance issue.

## Summary
The lab aesthetic is strong and coherent (acid-on-charcoal, mono labels, EM-### badges on panels), the 3D village is charming, the decision-trace inspector is genuinely excellent, and the console stayed clean for the whole session. The two big problems are: (1) **replay/inspector depth is an illusion** — every panel computes only from the client-side rolling history since page load, so scrubbing before that window shows a silently empty world (the backend `/api/replay`, `/api/events`, `/api/snapshots` all work; `inspectorApi` has zero consumers); and (2) **the sim quietly starved to extinction while looking cheerful** — needs/death pressure is invisible in the moment-to-moment UX and apparently in the agents' prompts too.

## Critical Issues

1. **Replay before the rolling window renders an empty black world with no explanation.** Scrubbed to tick 10 of a 95-tick run: "0 events at this tick", no agent dots, and every panel below (decision trace, governance, social graph, AWI) goes fully blank — no empty state, no "history not loaded" notice. The footer admits it: "Panels compute from the client-side rolling history (mock-safe, no backend)." Backend deep-replay works (`/api/replay?tick=10` returns a real snapshot + events; snapshots exist at ticks 0/25/50/75) but is never called. This breaks the #1 v2 promise ("see what happened").
2. **Mobile (375px) is unusable.** The center column (village + event feed — the product) is squeezed out entirely; left/right columns clip mid-button ("PA", truncated dropdowns). No responsive collapse exists. Tablet (768px) is acceptable.
3. **Total-extinction has no UX moment.** When the last agent died the sim just kept RUNNING (animal turns continue, header still green). No banner, no auto-pause, no "world over" summary. Death itself is handled (DEAD badges, reassign panel says "NO LIVE AGENTS") but the headline emergent event of the whole run passes silently.

## Improvements

4. **Duplicate `llm_call` rows in the decision trace.** Turn f32b85c8 (and others) shows two identical llm_call spans (#912/#913 — same tokens, same 2753.02ms latency). One call is being logged/rendered twice (W5 final-attempt logging + EM-067 per-attempt logging is the likely overlap). Makes token accounting look 2x.
5. **All three "different model" agents actually routed to the same model** (`→ ollama/cogito-2.1:671b` for gemini-flash, qwen-next, AND deepseek-pro). routed_via surfacing works, but nothing warns that the marquee multi-model experiment has silently collapsed to a single model. A "routing degraded: all profiles served by one model" banner would protect the experiment's validity.
6. **Energy crisis is invisible until death.** Ada/Bram sat at ENERGY 0 (red) for ~30+ ticks while their moods read "ecstatic"/"triumphant". Nothing escalates — no countdown ("dies in N turns"), no pulsing, no feed warning like "Ada is starving". The death rule (sustained zero) is opaque to the spectator.
7. **Replay agent-position map is nearly unreadable even in-window**: place squares are extremely low-contrast outlines; agent dots small; no labels. Compare with the main 2D map, which is clear.
8. **Inspector panels lack empty states everywhere** — Governance shows a vast black void with a tiny "ASSEMBLY IDLE"; decision-trace detail pane is blank until you click an entry (no hint); social graph empty area has no "no ties yet — relationships appear as agents interact" message (the AWI card does say "no ties yet", inconsistently).
9. **Animals are missing from the 2D world map** (and from the replay map legend semantics) — Mochi/Biscuit appear in the 3D village and feed but vanish on the 2D map toggle.
10. **Speed label / API mismatch:** control said "2.0s / tick" while `/api/state` reported `tick_interval_seconds: 3.0` (slider had been moved but state hadn't, or label reads slider-local state before server ack).
11. **Replay play/pause button label never changes** (always "PLAY") — `playingRef.current` in the render path (confirmed in code: `ReplayScrubber.tsx:319`).

## Minor Polish

12. Trace filter chip renders as disabled/ghost ("— Trace") with no tooltip explaining why (no trace events in window).
13. "TICK 83 · DAY 4" watermark clips at the bottom-right of the 2D map canvas.
14. Agent names truncate aggressively at ≤768px ("A…" for Ada) even when space exists in the card.
15. Heading semantics: almost everything is a `generic` div — panel titles ("EVENT STREAM", "PLAYBACK") should be real `h2`/`h3`s for screen readers (AWI panel does it right with `h3`s).
16. Model identity is encoded by color alone on village/map dots (color-vision users rely on the tiny text labels); a shape/initial fallback would help.
17. One dev-mode WS warning on load ("WebSocket closed before connection established") from the StrictMode double-connect — harmless but matches a real unmount-cleanup gap in `useSimulation`.

## What's Working Well
- The decision-trace chain UI (turn start → perceived → memory → llm call → reasoning → action → resolved → caused) is outstanding — readable, complete, with reflex/LLM tags and routed-via per call.
- Aesthetic coherence: the chaos-lab visual language is distinctive and consistent across both routes; EM-### badges on panels are a lovely touch.
- Death UX at the card level (DEAD badges, gray-out, reassign list pruning, "NO LIVE AGENTS") all updated live and correctly.
- Console: 0 errors across the entire session, both routes, including the WebGL unmount on `/inspector` (claim verified — no leaked render loop errors).
- a11y basics on controls are solid: every button/combobox/slider has a real accessible name; spawn form validates (button disabled until name).
- Animal chaos feed entries with in-character thoughts ("SO MANY LEGS TO CHASE OMG") read exactly as designed.
